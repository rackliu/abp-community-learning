# ABP 開源框架 教材設計：分級課程總覽

參考原始章節目錄：[`abp-community-learning-TOC.md`](abp-community-learning-TOC.md:2)

目標：建立覆蓋三個層級（入門 / 進階 / 實戰）的課程套件，包含章節教案、可執行範例、練習題庫與 PDF 匯出流程。

課程總體結構
- 入門（Foundation）：瞭解 ABP 核心概念、建立第一個專案、基本 CRUD 與權限
- 進階（Advanced）：深入模組化、DDD 應用、測試、效能優化
- 實戰（Practical）：微服務、CI/CD、容器化、SaaS 與企業案例整合

分章對應（採用原章節編號）

入門（建議學時：20-30 小時）
1. 現代軟體開發與 ABP Framework（Ch1）
    - 目標：理解 ABP 生態與架構理念；比較單層/分層/微服務選型
    - 成果：教材摘要、選型決策矩陣、快速入門範例專案
    - 時間：2 小時
2. ABP Framework 快速入門（Ch2）
    - 目標：使用 CLI/VS 建立並執行專案；理解 module 與 startup 流程
    - 成果：可運行的範例 repo、步驟化指令清單
    - 時間：4 小時
3. 實戰應用程式開發流程（Ch3）
    - 目標：掌握專案流程圖、開源版特有步驟
    - 成果：流程圖（Mermaid）、最小可行產品 (MVP) 指南
    - 時間：3 小時
4. 認識官方參考解決方案（Ch4）
    - 目標：導覽官方範例與社群套件
    - 成果：參考清單與延伸閱讀
    - 時間：2 小時

進階（建議學時：40-60 小時）
5. ASP.NET Core 與 ABP 架構深入解析（Ch5）
    - 目標：Module、依賴注入、開源模組擴充實作
    - 成果：範例模組、單元測試範本
    - 時間：6 小時
6. 資料存取基礎設施（Ch6）
    - 目標：Entity/Repository/UoW、EF Core 與 MongoDB 整合、多 DB 支援
    - 成果：多資料庫範例、遷移腳本
    - 時間：8 小時
7. 橫切關注 (Cross-Cutting Concerns)（Ch7）
    - 目標：權限、驗證、例外、日誌與審核實作
    - 成果：驗證流程與審計範例、測試案例
    - 時間：6 小時
8. 開源特色與服務功能（Ch8）
    - 目標：使用者管理、自動 API、分頁、快取、本地化
    - 成果：實作範例與設定範本
    - 時間：6 小時

實戰（建議學時：40-80 小時）
9-11. DDD 與領域層實作（Ch9-Ch11）
    - 目標：域模型、聚合、領域事件、DTO 與 Mapster 整合
    - 成果：以範例業務（例如訂單、客戶）實作聚合與事件
    - 時間：10-14 小時
12-13. 使用者介面與 API（Ch12-Ch13）
    - 目標：Razor/Blazor UI 設計、靜態/動態代理、前端資產管理
    - 成果：前端樣板、NPM 與 Bundling 範例
    - 時間：8-10 小時
14-15. 微服務與模組化開發（Ch14-Ch15）
    - 目標：建立微服務範本、Integration Service、API Gateway (YARP)
    - 成果：微服務範例專案、YARP 範例設定
    - 時間：10-16 小時
16. 多租戶與功能管理（Ch16）
    - 目標：多租戶架構、資料隔離、Feature Management 實作
    - 成果：多租戶範例與切換測試
    - 時間：4-6 小時
17. 測試與自動化（Ch17）
    - 目標：單元/整合測試、測試基礎設施建置
    - 成果：測試套件與 CI 範例
    - 時間：6-8 小時
18-19. 效能與安全（Ch18-Ch19）
    - 目標：Redis 快取、查詢優化、授權與資料保護
    - 成果：效能分析報告指引、安全稽核清單
    - 時間：6-8 小時
20-21. UI 現代化與主題客製（Ch20-Ch21）
    - 目標：React 基礎整合、LeptonX Lite / AdminLTE 主題客製（社群版）
    - 成果：React 範例與主題設定
    - 時間：6-8 小時
22-23. 部署、升級與遷移（Ch22-Ch23）
    - 目標：Docker/Helm/K8s、升級與遷移策略
    - 成果：部署腳本、升級檢核表
    - 時間：8-12 小時
24-25. 案例實踐與社群資源（Ch24-Ch25）
    - 目標：整合 EasyAbp 模組、設計企業級範例
    - 成果：完整示範專案、教學影片腳本
    - 時間：8-12 小時

評量與交付物
- 每章交付：Markdown 教案、範例程式碼、單元測試、練習題（含解答）
- 三個分級課程包：入門套件、進階套件、實戰套件（各自包含課時建議與評量）
- 最終匯出：可匯出 PDF（Pandoc/DocFX 流程），並上傳至 GitHub Releases

開發流程（課程製作里程碑）
- 草案（2 週）：完成所有章節大綱與學習目標
- 初版（6 週）：完成入門 + 50% 進階章節教案與範例
- 公開測試（2 週）：社群測試回饋並修正
- 正式發行（2 週）：完成全部章節、製作 PDF、發佈範例 repo

實作建議（Repository 結構）
- 建議採 monorepo：/docs (Markdown) /examples/{starter,modules,microservices} /scripts /ci
- 每章在 docs 對應一個目錄，examples 提供最小可運行的範例

風險與注意事項
- 避免包含 Pro/商業專屬功能；所有範例僅以社群可取得的模組為主
- 持續維護範例相容性（ABP 版本變動需標註）
